2.1  제품  개요

2.1.1  TnD-MIBS

[  유해  IP  차단  시스템  ]

㈜티앤디소프트의  MIBS(Malicious  IP  Blocking  System:  유해  IP  차단  시스템)은  다양한
배포기관으로부터  수집된  위협정보를  기반으로  위협이  존재하는  IP 로  접속하는  트래픽을
차단하여  잠재적인  해킹  위협으로부터  내부망을  보호하는  솔루션입니다.

[  도입  필요성  ]

-  다양한  경로를  통한  사이버  위협의  증가

-  유해사이트와  봇으로  인한  위협  IP  증가

-  사이버  공격  IP 의  재사용으로  인한  기존  위협  IP 에  대한  관리  필요

-  대용량  IP  차단  솔루션의  필요성

[제품  구성도]

다양한  위협  IP  배포  서버들로부터  전달받은  정보를  이용하여  각  기관  내/외부로  들어오고
나가는  패킷을  검사하여  대용량  IP  차단  시스템을  이용해  내부적으로  IP  차단을  진행합니다.

[제공  기능]

WSIS 는  아래와  같은  기능을  제공합니다.  기능별  자세한  설명은  2.2  절  이하를  참조하면
됩니다.

2.1.2  제품  특장점

다양한  배포기관에서  제공받는  위협  IP 를  이용하여  위협  IP 로의  접속을  원천  차단하여
잠재적인  해킹  위협으로부터  보호하기  위해  다음과  같은  MIBS 만의  특장점을  제공합니다.

[  대용량  IP  관리  시스템  ]

  표준  프로토콜  및  데이터  형식  지원  (TAXII/STIX)

사이버  위협  전송  규격인  TAXII 와  사이버  위협  표현  규격인  STIX 를  지원하는  인텔리전스
공유  플랫폼과의  연동이  가능합니다.

  중복  IP  처리

배포기관  별  또는  배포기간에  따라  중복된  위협  IP 를  제거하여  관리  대상  위협  IP  개수를
줄임으로써  메모리  절감  및  성능  개선을  지원합니다.  또한  중복  IP 를  처리하더라도
배포기관  정보는  유지하여  IP 에  대한  추적이  가능하도록  관리합니다.

  배포  기관  정보  저장

트래픽  차단  시  차단된  IP 에  대해  배포  기관  및  배포  일시  정보를  저장함으로써  추후  이력
추적이  가능합니다.

  예외  IP  관리

예외  IP  특정  기간  또는  차단시스템  전체/개별  적용  예외  IP  관리를  통해  효율적인  트래픽

  주기적인  위협  IP  배포

배포기관을  통해  전달받은  위협  IP 를  취합  후,  주기적으로  대용량  IP  차단  시스템으로

A-2.  차단  로그  활성화

추가  유해  IP  배포  설정  우측  아래에  존재하는  차단  로그  활성화는  추가  유해  IP  배포
