  버전  정보

-  API  문서  개정  이력에  명시된  버전을  사용

-  요청  URL에  버전  정보  포함

-

ex) http(s)://A.B.C.D:PORT/rest/v1.0/security-policy

  Request  기본  정보

-  Request header

-  Content-Type: application/x-www-form-urlencoded;

-  Cookie: SessionID=[SESSOINID]

-  로그인을  제외한  모든  요청은  Cookie에  SESSIONID를  포함하여야  함

-

SESSIONID는  로그인  시  응답  헤더에  포함되어  있음

-  예시

Connection: keep-alive

Content-Length: 53

Content-Type: application/x-www-form-urlencoded

Cookie: SessionID=66334873643C9869327B23C66B8B4567

-  Request parameter

-  GET, DELETE  방식: URL  파라미터로  정보를  받아  처리

-  URL  예시

http(s)://A.B.C.D:PORT/rest/v1.0/security-policy?id={id}

-

POST  방식: request body에  KEY=VALUE  형식으로  정보를  받아  처리

-

VALUE  는  JSON  포맷으로  입력

-  Request body  예시

data= {"username": "USERNAME", "password": "PASSWORD"}

-

KEY/VALUE는  문서에  정의된  타입에  따라  처리

-  타입이  BOOLEAN이고,  값이  false  인  경우  해당  KEY는  무시되며,

값이  true  경우맊  처리함

  Response  기본  정보

-  로그인  성공  시  SessionID를  header  정보에  포함하여  응답,  실패  시  403 Forbidden  코드로  응답

-  응답  메시지는  JSON  포맷으로  응답

-  잘못된  요청일  경우  404 Not found,  요청  처리  실패  시  400 Bad Request  코드로  응답

  JSON  메시지  정보

-  Request  포맷  내  KEY/VALUE  처리

-  VALUE의  타입이  BOOLEAN일  때  값이  false인  경우는  무시되며,  값이  true인  경우맊  처리됨

-

„set-first‟ KEY

-  설정  시  공통으로  포함되는  KEY

-

-

set first  가  false일  경우  엔트리  처리  방식을  unset->set  순으로  처리(기본  방식)

set-first  가  true일  경우  엔트리  처리  방식을  set->unset  순으로  처리(특히  마지막  엔트리

설정  변경일  경우  이  방식을  사용해야  됨)

-  각  항목  설정  조회  시  해당  KEY는  제외됨

-

„set‟, „unset‟ KEY

-  설정  시  공통으로  포함되는  KEY

-  엔트리  설정  추가일  경우  set  object  내  정의,  엔트리  설정  삭제일  경우  unset  object  내

정의

-  엔트리  설정  변경일  경우  unset,  set  둘다  정의되며,  순서는  „set-first‟  키  값에  따라  정해
